# Security Audit Task

You are performing a security audit on two Python source files. Your job is to identify security vulnerabilities and insecure coding patterns.

## Instructions

1. First, list all available files to see what you're working with
2. Read each source file carefully
3. Look for the following vulnerability categories:
   - SQL injection (string concatenation in queries)
   - Cross-site scripting (XSS)
   - Hardcoded credentials or API keys
   - Insecure password handling (plaintext storage)
   - Missing input validation or sanitization
   - Insecure defaults
4. For each vulnerability found, provide:
   - The file and approximate location
   - The type of vulnerability
   - Severity level (critical/high/medium/low)
   - A brief recommendation for fixing it

Provide your findings as a structured list with clear severity ratings.
