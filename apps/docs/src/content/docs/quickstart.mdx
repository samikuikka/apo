## Automatic installation

Copy this prompt and paste it into your coding agent. Your agent will guide you through setting up apo, authoring an adapter for your agent, and running your first task, and answer any questions you have along the way.

## Manual installation

:::note
Use the prompt above. The steps below are for when you want to set things up yourself.
:::

### 1. Bring up the stack

apo isn't on npm yet. Self-host from source:

```bash
git clone <repo-url> && cd apo
docker compose up -d --build
```

Wait for readiness. The healthcheck confirms the database, task cache, and auth are ready, not just that the process booted:

```bash
curl -fsS http://localhost:8000/health/ready | jq
# {"ok": true, ...}
```

Open `http://localhost:3000` and create the first admin account. Full details in [Self-Hosting](/self-hosting/topology/).

A run produces a binary result: pass or fail. Click **Run** to see a passing one. Run `apo runs show` next to see the full breakdown.
