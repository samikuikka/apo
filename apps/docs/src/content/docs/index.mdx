apo runs your real agent and reduces every task run to a pass or fail. When a run fails, you see which test broke and why, down to the message that caused it.

## Set up with an agent

apo is agent-installed. Point your coding agent at the setup prompt; it returns a markdown skill the agent follows end to end.

<CopyPrompt text={`Read https://apo.dev/start.md then help me set up apo and run my first task.`} />

The prompt lives at [`/start.md`](/start.md) (raw markdown). The steps are stubbed for now; the structure mirrors Flue's agent setup skill.
