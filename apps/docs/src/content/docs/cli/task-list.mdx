List the tasks available to run. With a `--project` set and the backend reachable, lists the backend's synced task inventory. Without one, scans the local `--dir` for `.eval.ts` files.

## Synopsis

```bash
apo task list
```

## Output

<TerminalOutput
  lines={[
    { text: "ID               Adapter      Checks  Simulator" },

    { text: "extract-parties  real-agent   yes     -" },
    { text: "code-review      real-agent   yes     -" },
    { text: "document-qa      real-agent   yes     -" },
    { text: "" },
    { text: "3 tasks found", tone: "default" },
  ]}
/>

The `ID` is the task folder name — pass it to [`apo task run`](/cli/task-run/). `Checks` is `yes` when the task registers any `test(...)` blocks, `-` when it has none.
