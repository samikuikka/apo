## Synopsis

```bash
apo runs show [run-id] [--verbose] [--exit-status]
```

## Description

Shows a run's details. The argument is a **run id** (the output of `apo task run`), a unique prefix of one, or the literal `last` (the default) for the most recent run.

Large per-check values — typically the deliverable re-sent inside every judged criterion's `received` — are shown as a one-line manifest (e.g. `+ Received: ⟨20,000 chars — apo runs deliverable⟩`) instead of dumped in full. This keeps a failed run readable: the `reasoning` explains the failure, not a wall of deliverable text, and it stops the same deliverable being re-billed per criterion. Read the full content with [`apo runs deliverable`](/cli/runs-deliverable/) — it fetches a deliverable **once**, not per check.

## Options

| Option | Default | Purpose |
|---|---|---|
| `[run-id]` | `last` | Run id, unique prefix, or `last`. With no arg, resolves the latest run. |
| `--verbose` | off | Show passing checks too (not just failures) and LLM-judge transcripts. |
| `--exit-status` | off | Exit non-zero if the run failed (for scripting). |
| `--task <id>` | — | When using `last`, scope to the latest run of this task. |

With `--verbose`, passing checks are listed with their matchers, and judged checks include the judge's model, prompt, and response — so you can see exactly what the judge was asked and what it said.

## The three ids

`apo runs show` takes a **run id**. This is distinct from the **task id** (`apo task run` takes that) and the **trace id** (a separate field on the run, printed at the bottom of the output — pass it to `apo traces show`).

## Example

```bash
# Show the latest run
apo runs show

# Show a specific run by id
apo runs show de89cab0

# Show with passing checks and judge transcripts
apo runs show --verbose

# Scope "last" to a specific task
apo runs show last --task extract-parties
```
