## `apo batch create`

Create a new batch run on the backend.

### Synopsis

```bash
apo batch create --tasks <id1,id2,...>
```

### Options

| Option | Default | Description |
|---|---|---|
| `--tasks <id1,id2,...>` | (required) | Comma-separated task ids. |

Without a `--project`, the ids are resolved against local `--dir` discovery and the filesystem paths are sent. With a `--project`, the ids are sent directly.

## `apo batch show`

Show a batch run's details — aggregate counts, per-task results, costs, and trace ids.

### Synopsis

```bash
apo batch show <batch-id> [--watch]
```

### Options

| Option | Default | Description |
|---|---|---|
| `<batch-id>` | (required) | Batch id or unique prefix. |
| `--watch` | off | Clear the screen and refresh every 3s while the batch is `running` or `queued`. |

