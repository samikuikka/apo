Project commands manage which project you're operating against and where its tasks come from. Every command respects the global `--project` flag (or the saved project from `apo login`).

## `apo project list`

List projects you can access. The active project is marked with `*`.

### Synopsis

```bash
apo project list
```

## `apo project use`

Switch the active project. Rewrites `~/.apo/credentials` with the new project id.

### Synopsis

```bash
apo project use [id|name|prefix]
```

With no argument and more than one project, opens an interactive picker. Accepts an id, name, or unique prefix as a positional or via `--project`.

### Output

<TerminalOutput
  lines={[
    { text: "✓ Switched to acme-evals (a1b2c3d4)", tone: "success" },
    { text: "  Run `apo project list` to see all your projects.", tone: "default" },
  ]}
/>

### Examples

```bash
apo project use acme-evals
apo project use staging    # unique prefix
apo project use            # interactive picker
```

## `apo project init-tasks`

Configure and sync a GitHub-backed task source in one step. This is the fastest path from zero to runnable tasks: it sets the source, triggers a sync, and — if GitHub isn't connected yet — opens the browser for OAuth and retries.

### Synopsis

```bash
apo project init-tasks --repo <owner/repo | url> [--branch <name>] [--subpath <path>] [--name <text>]
```

### Options

| Option | Default | Description |
|---|---|---|
| `--repo <owner/repo \| url>` | (required) | GitHub repository. Accepts `owner/repo` or a full `github.com/...` URL. |
| `--branch <name>` | `main` | Branch to clone. |
| `--subpath <path>` | repo root | Subdirectory within the repo where tasks live. |
| `--name <text>` | `owner/repo` | Display name for the task source. |

### What it does

1. Normalizes the repo input to a clone URL.
2. `PATCH`es the project's task source to type `git`.
3. Attempts a sync. If it fails and GitHub OAuth isn't connected, opens the browser for authorization, polls until connected (up to 3 minutes), then retries.
4. Lists the discovered task count.

```bash
apo project init-tasks --repo acme/agent-evals --branch main --subpath e2e/tasks
```

## `apo project source`

Manage the task source — where tasks come from. Three subcommands:

### `apo project source show`

Prints the configured source: type (git/filesystem/demo), repository, ref, subpath, status, last synced commit, and any error.

#### Synopsis

```bash
apo project source show
```

### `apo project source set`

Create or replace the task source.

```bash
apo project source set --type <git|filesystem|demo> [options]
```

| Option | Required for | Description |
|---|---|---|
| `--type <git\|filesystem\|demo>` | all | Source type. |
| `--repo <url>` | git | Clone URL. |
| `--ref <branch>` | git | Branch (default `main`). |
| `--subpath <path>` | — | Subdirectory where tasks live. |
| `--path <dir>` | filesystem | Local directory to scan. |
| `--name <text>` | — | Display name. |
| `--seed <id>` | demo | Demo seed id (default `default`). |

### `apo project source sync`

Sync the configured source into the task inventory without listing the tasks. Use [`apo project sync-tasks`](#apo-project-sync-tasks) if you also want the count.

#### Synopsis

```bash
apo project source sync
```

## `apo project sync-tasks`

Sync the task inventory and report the count. Combines `source sync` with a task listing so you see what landed.

### Synopsis

```bash
apo project sync-tasks
```
