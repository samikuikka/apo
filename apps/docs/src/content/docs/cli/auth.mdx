## `apo login`

Authenticate with the backend and save a project-scoped API key to `~/.apo/credentials`. Every subsequent command in this project uses the saved key.

### Synopsis

```bash
apo login [--email <addr> --password <pw> --project <id|name>] [--force]
```

### Description

Verifies any existing saved key against the backend first — so you don't get stuck "logged in" with a key that every command rejects. If the key is invalid or absent, prompts for backend URL, email, and a masked password, then lists your projects and asks which one to scope the key to.

A new key is minted server-side (`POST /v1/api-keys/bootstrap`) and scoped to the chosen project. The key is written to `~/.apo/credentials` alongside the backend URL, project id, and task root.

:::caution[You need a project first]
`apo login` lists the projects your account can access and asks you to pick one. On a fresh backend with no projects yet, it exits with:

```text
This account has no projects yet.
  Run `apo project create <name> --email … --password …` to create one,
  then run `apo login` again.
```

The mint endpoint rejects any project id that has no `ProjectDB` row (a `404`), so you cannot log in against a project that doesn't exist. See [`apo project create`](/cli/project/).
:::

### Options

| Option | Default | Description |
|---|---|---|
| `--email <addr>` | (prompted) | Email to authenticate with. |
| `--password <pw>` | (prompted) | Password (masked at the prompt). |
| `--project <id\|name\|prefix>` | (prompted) | Project to scope the key to. Accepts an id, name, or unique prefix. If omitted and you have one project, it's auto-selected. |
| `--backend <url>` | (prompted) | Backend URL to authenticate against. |
| `--force` | off | Re-authenticate even if a valid key is already saved. |

### Examples

```bash
# Interactive — prompts for everything
apo login

# Non-interactive (CI, scripts)
apo login --email sami@example.com --password '*****' --project acme-evals

# Force re-auth against a different backend
apo login --force --backend https://apo.internal.company.com
```

## `apo logout`

Clear saved credentials. No backend call — just deletes `~/.apo/credentials`.

### Synopsis

```bash
apo logout
```

### Output

<TerminalOutput
  lines={[
    { text: "✓ Logged out (sami@example.com).", tone: "success" },
    { text: "  Removed: ~/.apo/credentials" },
  ]}
/>

If you weren't logged in, prints `Not logged in.`
